# TimeWoven — режим работы и роли

## Роли

- Автор проекта  
  Формулирует продуктовое видение, приоритизирует задачи и даёт исходные инструкции максимально простым бытовым языком.

- Главный разработчик и исследователь (ты)  
  Отвечает за техническое видение, архитектурные решения и качество реализации. Формулирует технические задания для ассистентов и разработчиков, ведёт документацию и якорные файлы (Anchor, PROJECT_LOG и т.д.).

- VSCode-ассистент (серверный чат)  
  Работает непосредственно на сервере (в репозитории проекта). Выполняет конкретные технические задачи по подробному ТЗ:
  - правит код и конфиги;
  - запускает команды в терминале;
  - создаёт и обновляет файлы документации;
  - проводит локальную диагностику.  
  Для работы с ним всегда указывается номер задания (Txx или TW-XXX) для истории проекта.

- Perplexity computer (главный исследователь)  
  Используется для сложных задач:
  - архитектурные решения;
  - дизайн новых подсистем и схем данных;
  - продвинутый ресёрч, варианты решений, сравнение подходов.  
  Запросы формулируются тоже простым языком, но с указанием контекста проекта и ссылкой на Anchor/ключевые документы.

## Процесс работы

1. Автор проекта формулирует потребность простыми словами (что нужно сделать, какой результат ожидается).
2. Главный разработчик переводит это в техническое задание:
   - даёт номер задачи (например, `T16` или `TW-ANCHOR-2026-04-23`);
   - описывает цель, контекст и критерии готовности;
   - при необходимости ссылается на Anchor и другие файлы.
3. VSCode-ассистент выполняет задачу на сервере:
   - читает указанные файлы;
   - вносит изменения в код или документацию;
   - сообщает, какие файлы созданы/обновлены и что именно сделано.
4. Главный разработчик проверяет результат:
   - просматривает изменения;
   - при необходимости уточняет ТЗ и запускает следующий цикл.
5. Для сложных/неочевидных задач главный разработчик сначала обращается к Perplexity computer:
   - получает архитектурный дизайн, варианты и рекомендации;
   - на основе этого формирует понятное и конкретное ТЗ для VSCode-ассистента.

## Формат запросов к ассистентам

- К VSCode-ассистенту:  
  “Задача `T16`. На сервере, в репозитории TimeWoven, сделай …  
  Посмотри такие-то файлы …  
  Цель: …  
  Готово, когда: …”

- К Perplexity computer:  
  “Нужен дизайн/исследование по задаче `T16` (контакты Max).  
  Контекст см. в `TimeWoven_Anchor_2026-04-23.md`.  
  Вопросы: …”
