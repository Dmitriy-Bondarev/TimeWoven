# TimeWoven Documentation

Документация проекта разделена на уровни:

---

## Core

Ключевые архитектурные документы системы.

* TECH_PASSPORT.md — архитектура и модель данных
* PROJECT_OPS_PROTOCOL.md — правила внесения изменений (главный документ)
* AI_INIT.md — правила работы AI-архитектора

👉 Читать в первую очередь

---

## ADR (Architecture Decision Records)

История архитектурных решений проекта.

* ADR-001 … ADR-006 — зафиксированные решения
* объясняют, почему система устроена именно так

---

## Project

Операционная документация и история проекта.

* PRODUCT_BACKLOG.md — задачи
* PROJECT_LOG.md — принятые решения
* CHANGELOG.md — изменения системы
* DB_CHANGELOG.md — изменения БД
* FAMILY_GRAPH_ANALYSIS.md — аналитика
* anchors/ — контрольные состояния проекта

---

## Database

Документы, связанные со структурой базы данных.

* DATABASE_SCHEMA.md — описание структуры
* create_postgres_schema.sql — базовая схема

---

## Research

Исследовательские и промежуточные материалы.

* используются для проработки идей
* не являются источником истины

---

## Snapshots

Технические снимки состояния системы.

* используются для диагностики и восстановления контекста

---

## Принцип

* Core → как устроена система
* ADR → почему она так устроена
* Project → как она развивается
* Database → как хранится
* Research → куда думаем
* Snapshots → что было зафиксировано
