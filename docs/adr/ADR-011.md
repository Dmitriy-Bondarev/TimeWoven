# ADR-011: User Layer v1 and Membership Model

**Status:** Proposed  
**Date:** 2026-04-29  
**Related:** ADR-010 (User Layer Foundation)

---

## Context

Current system uses UUID family access + TOTP.  
Need scalable ownership model.

---

## Decision

### Entities

- users
- families
- family_memberships
- invites
- consents
- audit_log

### Relations

- one user may belong to many families
- one family may have many users

### Roles

- owner
- editor
- viewer

### Compatibility

Current UUID + TOTP family login remains valid.

### Future

- Email login
- Magic links
- Multi-device accounts
- Mobile apps
- GDPR deletion flow

---

## Consequences

- No breaking auth changes now.
- Enables future migrations.
