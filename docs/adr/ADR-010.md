# ADR-010: User Layer Foundation

**Status:** Proposed  
**Date:** 2026-04-29  
**Related:** ADR-009 (User Ownership and GDPR Compliance Foundation), ADR-006 (Temporal Layers and Snapshot Navigation for Family Graph)

---

## Context

TimeWoven currently stores family data but lacks explicit user ownership model.

---

## Decision

Introduce:

- users
- family_memberships
- roles: owner/editor/viewer
- invites
- consent tracking
- audit trail

Ownership rule:

User owns workspace.  
Workspace contains one family tree initially.

Future-ready:  
one user may own multiple family spaces.

---

## Consequences

Enables:

- login system
- GDPR rights
- access sharing
- mobile apps
- subscriptions
- admin moderation

---

## Notes

Implementation split into future phases.
