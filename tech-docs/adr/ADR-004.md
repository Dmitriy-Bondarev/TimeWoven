# ADR-004: Mac Audio Watcher — архитектура клиентской части пипелайна транскрипции

**Дата:** 2026-04-18  
**Статус:** Принято  
**Автор:** Dmitriy-Bondarev  

---

## Контекст

Проект TimeWoven включает функцию транскрипции аудио-воспоминаний (рассказы родственников). Администратор записывает аудио на своём Mac и хочет, чтобы файлы автоматически попадали в систему для транскрипции.

## Решение

Realise lightweight polling-based watcher (`scripts/watcher.py`) для macOS, без установки тяжёлых ML-моделей.

## Мотивация

- Администратор работает с Mac, Whisper слишком тяжёлый для скачивания на мобильном интернете
- Транскрипция должна выполняться на сервере (где есть GPU/CPU ресурс)
- Необходимо избежать повторной загрузки одних и тех же файлов

## Архитектура

```
Mac (watcher.py)
    │
    ├── Следит за ~/Desktop/timewoven_audio/ (polling 10с)
    ├── SHA256 fingerprint — избежаем дубликаты
    ├── POST /api/audio/upload → Сервер
    └── Лог ~/.timewoven_watcher_log.json

Server (app.timewoven.ru)
    ├── /api/audio/upload — принимает файл, сохраняет в DB
    ├── Whisper (вечером) — транскрибирует файл
    └── /api/transcription/result — возвращает текст
```

## Рассмотренные альтернативы

| Вариант | Причина отказа |
|--------|------------------|
| Whisper локально на Mac | Тяжёлая загрузка на мобильном интернете |
| watchdog библиотека | Избыточная зависимость, polling достаточно |
| inotifywait | Только Linux |

## Последствия

- Вечером: добавить Whisper на сервер, подключить `/api/audio/upload`
- Опционально: перевести в режим fsevents (без polling) если задержки будут критичными
