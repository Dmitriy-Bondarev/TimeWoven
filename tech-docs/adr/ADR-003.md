# ADR-003: Аудио пайплайн и транскрипция через Whisper

**Дата:** 2026-04-18  
**Статус:** Принято  
**Автор:** Dmitriy Bondarev  

---

## Контекст

TimeWoven хранит аудиозаписи воспоминаний членов семьи. Необходима автоматическая транскрипция аудио в текст для дальнейшего поиска, отображения и редактирования. Текущий VPS (1.8GB RAM) не позволяет запустить Whisper локально.

## Решение

Гибридная архитектура: VPS обрабатывает веб-запросы, MacBook Pro M5 Pro (48GB RAM) выполняет транскрипцию.

### Статусы транскрипции

| Статус | Описание |
|--------|----------|
| `pending` | Аудио загружено, ожидает обработки |
| `processing` | Mac выполняет транскрипцию |
| `review` | Текст готов, ожидает проверки администратора |
| `published` | Проверено администратором, доступно пользователю |
| `error` | Ошибка при транскрипции |

## Обоснование

- **MacBook Pro M5 Pro 48GB** — в разы мощнее любого VPS, Whisper large-v3 обрабатывает 10 мин аудио за ~15 сек через Apple Neural Engine / MLX
- **Whisper large-v3 локально** — лучшее качество для русского языка, старых голосов, шума; данные не покидают инфраструктуру (vs OpenAI API)
- **Шаг проверки администратором** — обязателен на старте для контроля качества; будет убран когда качество транскрипции подтвердится как стабильное

## Альтернативы, которые были отклонены

| Вариант | Причина отклонения |
|---------|-------------------|
| OpenAI Whisper API | Данные семьи уходят во внешний сервис, зависимость от интернета |
| Whisper на VPS (CPU) | 1.8GB RAM недостаточно; на CPU ~10-15 мин на 1 мин аудио |
| Апгрейд VPS до medium | Нецелесообразно при наличии M5 Pro |

## Планируемая эволюция

- **v1.0 (сейчас):** ручной запуск watcher.py на Mac + проверка администратором
- **v1.1:** автоматический watcher.py как launchd-сервис на Mac
- **v2.0:** при росте >100 активных пользователей — переход на VPS medium + Whisper на сервере, шаг проверки администратором убирается

## Последствия

- Требуется реализовать: `POST /api/transcription/result`, скрипт `watcher.py`, страница `/admin/transcriptions`, поле `transcription_status` в таблице Memories
- MacBook Pro должен быть доступен для обработки (не критично — очередь ждёт)
- При переезде на новый сервер — архитектура не меняется
