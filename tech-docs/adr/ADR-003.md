ADR-003: Аудио пайплайн и транскрипция через Whisper
Дата: 2026-04-18
Статус: Принято
Автор: Dmitriy Bondarev

## Контекст
TimeWoven хранит аудиозаписи воспоминаний членов семьи. Необходима автоматическая транскрипция аудио в текст для поиска, отображения и редактирования. Текущий VPS (1.8GB RAM) не позволяет запустить Whisper локально.

## Решение
Гибридная архитектура на старте: VPS обрабатывает веб-запросы, MacBook Pro M5 Pro (48GB RAM) выполняет транскрипцию.

## Пайплайн v1.0
Пользователь -> загружает аудио
VPS -> /static/audio/raw/ -> статус БД: "pending"
MacBook Pro M5 Pro -> забирает файл (rsync/scp)
Whisper large-v3 (MLX) -> транскрибирует
MacBook -> POST /api/transcription/result -> статус: "review"
Администратор -> /admin/transcriptions -> проверяет/редактирует -> "Опубликовать"
VPS -> статус: "published" -> пользователь видит текст

## Статусы транскрипции
Статус	Описание
pending	Аудио загружено, ожидает обработки
processing	Mac выполняет транскрипцию
review	Текст готов, ожидает проверки администратора
published	Проверено администратором, доступно пользователю
error	Ошибка при транскрипции
Обоснование
MacBook Pro M5 Pro 48GB — мощнее любого VPS, Whisper large-v3 обрабатывает 10 мин аудио за ~15 сек через Apple Neural Engine / MLX

Whisper large-v3 локально — лучшее качество для русского языка, старых голосов, шума; данные семьи не покидают инфраструктуру

Шаг проверки администратором — обязателен на старте; убирается после подтверждения стабильной работы

## Альтернативы, которые были отклонены
Вариант	Причина отклонения
OpenAI Whisper API	Данные семьи уходят во внешний сервис
Whisper на текущем VPS (CPU)	1.8GB RAM недостаточно, скорость неприемлема
Апгрейд VPS до medium сейчас	Нецелесообразно при наличии M5 Pro
Планируемая эволюция
v1.0 (сейчас): watcher.py на Mac вручную/полуавтоматически + проверка администратором

v1.1: watcher.py как launchd-сервис; проверка убирается после подтверждения качества

v2.0 (при росте пользователей): полный переезд на мощный VPS — вся обработка включая Whisper переносится на сервер; MacBook исключается из пайплайна; архитектура полностью серверная и независимая от локального устройства

## Требует реализации
POST /api/transcription/result — эндпоинт для приёма результата транскрипции с Mac

watcher.py — скрипт на Mac: мониторинг новых файлов, запуск Whisper large-v3, отправка результата

/admin/transcriptions — страница проверки и редактирования транскриптов администратором

Поле transcription_status в таблице Memories (pending / processing / review / published / error)
